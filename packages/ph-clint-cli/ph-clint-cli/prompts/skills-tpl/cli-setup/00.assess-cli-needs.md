# Phase 0: Assess CLI Needs

Before writing any code, determine which features the CLI requires.

## Steps

### 1. Name the CLI

Choose a kebab-case name. This name is used for:
- Config file paths: `{workdir}/.ph/{name}.config.local.json`
- Environment variable prefix: `{NAME}_*` (UPPER_SNAKE_CASE)
- Package `"bin"` field and executable name

### 2. Determine version sourcing

Choose one:
- **Hardcoded**: `version: '1.0.0'`
- **From package.json**: `const { version } = readPackageInfo(import.meta.url)`

### 3. Feature checklist

For each feature, determine if the CLI needs it:

- [ ] **Custom config?** → needs `configSchema` (Zod object)
- [ ] **Sensitive config (API keys, tokens)?** → needs `secretsSchema` (Zod object)
- [ ] **Interactive REPL?** → needs `interactive` with `welcome` message
- [ ] **Background services?** → needs `services` array of `defineService()` definitions
- [ ] **Automated triggers?** → needs `triggers` array + `routine` config
- [ ] **AI agent?** → needs `cli.configureAgent(loader)` after `defineCli()`
- [ ] **Skill templates?** → needs `prompts` config with `artifacts`, `agents`, `skills`
- [ ] **Powerhouse documents?** → needs `cli.configureReactor(config)` after `defineCli()`
- [ ] **Service lifecycle events?** → needs `events` handlers
- [ ] **Implementation-level workdir override?** → needs `workdir` option (hides `--workdir` flag)

### 4. List commands to register

Enumerate all commands that will be passed to the `commands` array. These should already be defined using `defineCommand()` (see the `command-definition` skill). Note which ones need agent-compatible `{ text, data }` returns.

### 5. Detect codegen vs manual project

For existing projects, check if `cli.ts` already has `@clint:begin` marker regions:
- **Codegen-managed**: Markers present — update `project-spec.json` and run `{{commands.clint-project-regen.id}}`. Never hand-edit marker regions.
- **Manual**: No markers — edit `defineCli()` directly as a regular TypeScript file.

For new projects, prefer using `{{commands.clint-project-init.id}}` to generate the scaffold with markers.

### 6. Determine config field names

For each config/secret field, verify the auto-generated env var name is sensible:
- Field `apiKey` on CLI `mycli` → env var `MYCLI_API_KEY`
- Field `connectPort` on CLI `vetra` → env var `VETRA_CONNECT_PORT`

If a name would be confusing, consider renaming the field.
