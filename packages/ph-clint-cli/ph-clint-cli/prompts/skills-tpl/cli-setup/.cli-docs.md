## API Reference: CLI Setup

### `defineCli<TSchema, TSecrets>(options): Cli`

Factory function that creates a CLI from a declarative options object. Two generic type parameters:

- `TSchema extends z.ZodType` — config schema type (defaults to `Record<string, unknown>`)
- `TSecrets extends z.ZodType` — secrets schema type (defaults to `Record<string, unknown>`)

### `CliOptions<TSchema, TSecrets>` Interface

```typescript
interface CliOptions<TSchema, TSecrets> {
  name: string;                          // CLI name (kebab-case, used for config files and env var prefix)
  version: string;                       // Semantic version (shown in --version)
  root?: string;                         // Absolute path to package root (resolves sibling packages)
  description: Resolvable<string, Config>; // Static string or function of ({ config, workdir })
  commands: Command<any, any, any>[];    // Array of command definitions
  configSchema?: TSchema;                // Zod schema for user-facing config
  secretsSchema?: TSecrets;              // Zod schema for sensitive config (API keys, tokens)
  interactive?: InteractiveConfig<Config>; // REPL configuration
  triggers?: Trigger<any, any, any>[];   // Routine loop triggers
  routine?: RoutineConfig;               // Routine loop timing & project scanner
  services?: ServiceDefinition<Config>[]; // Background service definitions
  events?: Record<string, (data: any, log: Logger) => void>; // Event handlers
  workdir?: string;                      // Implementation-level workdir override (hides --workdir flag)
  configDefaults?: Record<string, unknown>; // Layer 5 config defaults
  prompts?: PromptsConfig;               // Agent skills & profiles
}
```

### `InteractiveConfig<TConfig>` Interface

```typescript
interface InteractiveConfig<TConfig> {
  welcome: Resolvable<string, TConfig>;  // Welcome message (static or dynamic)
  outputWindow?: number;                 // Trailing output lines in REPL (default: 6)
}
```

### `Resolvable<T, TConfig>` Type

```typescript
type Resolvable<T, TConfig> = T | ((ctx: { workdir: string; config: TConfig }) => T);
```

Allows static values OR dynamic computation based on resolved workdir and config.

### `Cli` Object Interface

Methods available on the object returned by `defineCli()`:

| Method | Signature | Description |
|--------|-----------|-------------|
| `run` | `(argv: string[], options?: RunOptions) => Promise<void>` | Main entry point — parse args, dispatch command or start REPL |
| `bootstrap` | `(opts?: BootstrapOptions) => Promise<BootstrapResult>` | Initialize runtime without dispatch — for programmatic access |
| `execute` | `(commandId, args, context?) => Promise<unknown>` | Execute a command directly (programmatic use) |
| `parseArgs` | `(commandId, argv) => Record<string, unknown>` | Parse CLI argv for a specific command |
| `getCommand` | `(id: string) => Command \| undefined` | Look up a registered command |
| `listCommands` | `() => Command[]` | List all registered commands |
| `generateHelp` | `() => string` | Generate CLI help text |
| `generateCommandHelp` | `(commandId: string) => string` | Generate help for a specific command |
| `generateCompletion` | `(shell: string) => string` | Generate shell completion script (bash/zsh/fish) |
| `configEnvVars` | `() => ConfigEnvVar[]` | List config env var mappings |
| `getMetadata` | `() => CliMetadata` | JSON-serializable CLI metadata |
| `configureAgent` | `(loader: AgentLoader<any>) => void` | Set the agent factory (lazy-loaded) |
| `configureReactor` | `(config: ReactorConfiguration) => void` | Set the reactor configuration (lazy-loaded) |

Properties:

| Property | Type | Description |
|----------|------|-------------|
| `name` | `string` | CLI name |
| `version` | `string` | CLI version |
| `description` | `string` | CLI description |
| `hasAgent` | `boolean` | True when `configureAgent()` has been called |
| `hasReactor` | `boolean` | True when `configureReactor()` has been called |

### `RunOptions` Interface

```typescript
interface RunOptions {
  exit?: (code: number) => void;         // Override process.exit (for testing)
  stdout?: (message: string) => void;    // Capture stdout
  stderr?: (message: string) => void;    // Capture stderr
  interactiveInput?: AsyncIterable<string>; // Headless REPL input (for testing)
  resume?: string;                       // Resume agent conversation by thread ID
  workdir?: string;                      // Override workdir (for testing)
  configFile?: string;                   // --config flag value
  logLevel?: LogLevel;                   // Override log verbosity (default: 'info')
}
```

### `BootstrapOptions` & `BootstrapResult`

```typescript
interface BootstrapOptions {
  workdir?: string;        // Explicit workdir (skips flag/cwd logic)
  configFile?: string;     // Config file override
  logLevel?: LogLevel;     // Log level (default: 'info')
  stdout?: (text: string) => void;
  stderr?: (text: string) => void;
}

interface BootstrapResult {
  workdir: string;
  config: Record<string, unknown>;
  context: CommandContext;
  getAgent(): Promise<AgentProvider | undefined>;
  get mastraAgent(): Promise<unknown | undefined>;
}
```

### Built-in Commands (Auto-Injected)

| Command ID | Condition | Purpose |
|-----------|-----------|---------|
| `cli-docs` | Always | Show CLI help (also usable as agent tool) |
| `config` | When `configSchema` provided | View/modify configuration settings |
| `{service-id}-start` | Per service definition | Start a background service |
| `{service-id}-stop` | Per service definition | Stop a running service |
| `{service-id}-restart` | Per service definition | Restart a service |
| `{service-id}-ps` | Per service definition | Show service status |
| `{service-id}-logs` | Per service definition | Tail service logs |
| `{service-id}-manage` | Per service definition | Interactive management (REPL only) |
| `{service-id}-ls` | Per service + projectScanner | List discovered projects |

### `readPackageInfo(importMetaUrl)`

Utility to read name and version from the nearest `package.json`:

```typescript
import { readPackageInfo } from '@powerhousedao/ph-clint';
const { name, version } = readPackageInfo(import.meta.url);
```
