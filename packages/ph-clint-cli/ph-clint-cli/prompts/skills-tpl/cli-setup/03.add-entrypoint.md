# Phase 3: Create the Entry Point

Set up the CLI entry point and package.json configuration.

## Steps

### 1. Create the entry point file

Create `src/cli.ts` (or `src/main.ts`) with the Node.js shebang:

```typescript
#!/usr/bin/env node
import { defineCli } from '@powerhousedao/ph-clint';
import { configSchema } from './config.js';
import { myCommand } from './commands/my-command.js';

const cli = defineCli({
  name: 'my-app',
  version: '1.0.0',
  description: 'My application',
  commands: [myCommand],
  configSchema,
  interactive: {
    welcome: 'My App — type /help for commands',
  },
});

cli.run(process.argv);
```

If the CLI definition is complex, keep it in a separate file (`src/cli-definition.ts`) and import it:

```typescript
#!/usr/bin/env node
import { cli } from './cli-definition.js';
cli.run(process.argv);
```

### 2. Configure package.json

Add these fields:

```json
{
  "type": "module",
  "bin": {
    "my-app": "./dist/cli.js"
  },
  "scripts": {
    "build": "tsc",
    "start": "node dist/cli.js",
    "dev": "tsx src/cli.ts"
  }
}
```

- `"type": "module"` — required for ESM imports
- `"bin"` — maps the CLI name to the compiled entry point
- `"dev"` script uses `tsx` for development without compilation

### 3. Configure tsconfig.json

Ensure these compiler options:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "Node16",
    "moduleResolution": "Node16",
    "outDir": "./dist",
    "rootDir": "./src",
    "declaration": true,
    "strict": true,
    "esModuleInterop": true
  },
  "include": ["src"]
}
```

### 4. Add ph-clint dependency

```sh
pnpm add @powerhousedao/ph-clint zod
```

Or for local development with the library:

```json
{
  "dependencies": {
    "ph-clint": "file:../../packages/ph-clint"
  }
}
```

### 5. Codegen projects already have entry points

In codegen-managed projects (created by `{{commands.clint-project-init.id}}`), the scaffold already includes:
- `src/main.ts` — shebang entry point with `cli.run(process.argv)`
- `src/cli.ts` — `defineCli()` with marker regions
- `src/framework.ts` — user-owned config schemas and `createTypes()`
- `package.json` — pre-configured with `bin`, `type: "module"`, and scripts

Skip steps 1-4 if these already exist. Proceed directly to verification.

### 6. Verify the CLI works

Run these checks:

```sh
# Build
pnpm build

# Help output (should show all commands)
node dist/cli.js --help

# Version
node dist/cli.js --version

# A specific command
node dist/cli.js my-command --help

# Interactive mode (if configured)
node dist/cli.js -i
```

Expected: `--help` shows the description, version, all registered commands (including auto-injected ones like `config` and `cli-docs`), and global options (`-i`, `--workdir`, `--config`, `--verbose`).
