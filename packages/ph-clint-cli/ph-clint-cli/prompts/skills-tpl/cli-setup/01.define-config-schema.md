# Phase 1: Define Config and Secrets Schemas

Design and wire the configuration layer. Skip this phase if the CLI needs no configuration.

## Steps

### 1. Define configSchema

Create a Zod object schema for user-facing settings:

```typescript
import { z } from 'zod';

export const configSchema = z.object({
  model: z.string().default('anthropic/claude-sonnet-4-5').describe('LLM model to use'),
  port: z.number().default(4001).describe('Server port'),
  verbose: z.boolean().default(false).describe('Enable verbose output'),
});
```

Rules:
- Every field MUST have `.describe('...')` — it appears in `config` command output, `--help`, and env var listings
- Use `.default(value)` for fields with sensible defaults
- Use `.optional()` only for truly optional fields with no default
- Prefer specific types: `z.enum([...])` for constrained choices, `z.number()` for ports

### 2. Define secretsSchema

Create a separate Zod object schema for sensitive values:

```typescript
export const secretsSchema = z.object({
  apiKey: z.string().optional().describe('LLM API key'),
  dbToken: z.string().optional().describe('Database access token'),
});
```

Rules:
- Secrets are almost always `.optional()` — the CLI should work without them (possibly in a degraded mode)
- These fields are merged into configSchema internally but censored in output
- They follow the same env var naming: `{CLI_NAME}_{FIELD}`

### 3. Set configDefaults (if needed)

Use `configDefaults` in `defineCli()` for implementation-specific overrides that differ from the schema defaults:

```typescript
defineCli({
  configSchema,
  configDefaults: {
    port: 8080, // This implementation uses 8080, not the schema default of 4001
  },
});
```

This is layer 5 — above schema `.default()` but below user/local config files.

### 4. Use createTypes for typed commands

If you have a configSchema, create a typed factory so all commands get typed `context.config`:

```typescript
// framework.ts
import { createTypes } from '@powerhousedao/ph-clint';
import { configSchema, secretsSchema } from './config.js';

export const { defineCommand, defineTrigger } = createTypes({ configSchema, secretsSchema });
export { configSchema, secretsSchema };
```

Then import `defineCommand` from `./framework.js` instead of `ph-clint` in all command files.

### 5. Verify env var names

Run through each field and check the auto-generated env var name:
- `{CLI_NAME}` is the CLI name uppercased with hyphens replaced by underscores
- Field keys are converted to UPPER_SNAKE_CASE
- Example: CLI `my-app`, field `connectPort` → `MY_APP_CONNECT_PORT`
