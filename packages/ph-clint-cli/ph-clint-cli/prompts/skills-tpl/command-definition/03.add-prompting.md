# Phase 3: Configure Interactive Prompting (Optional)

Set up parameter prompting for REPL/interactive mode.

## When to Add Prompting

Add prompting when the command is frequently used in interactive mode and has optional parameters that benefit from guided input. Skip this phase if the command is primarily used in command mode or all parameters are required.

## Steps

### 1. Add `prompt` config to `defineCommand`

```typescript
export const addTask = defineCommand({
  id: 'add-task',
  description: 'Add a new task',
  inputSchema: z.object({
    title: z.string().describe('Task title'),
    priority: z.enum(['low', 'medium', 'high']).optional().describe('Task priority'),
    due: z.string().optional().describe('Due date (YYYY-MM-DD)'),
  }),
  prompt: {
    promptForDefaults: false,
    promptOptional: ['priority'],
  },
  execute: async ({ title, priority, due }, { workspace }) => {
    // ...
  },
});
```

### 2. Understand the prompting options

**`promptForDefaults: false`** (recommended):
- Fields with `.default()` values will NOT be prompted for
- The default value is used silently
- Set to `true` only if users frequently need to override defaults

**`promptOptional: ['field1', 'field2']`**:
- These `.optional()` fields WILL be prompted for in REPL mode
- Omitted optional fields are NOT prompted (they stay `undefined`)
- Use this for fields that are useful but not required

### 3. How prompting works at runtime

- **CLI mode** (`mycli add-task --title "Buy milk"`): No prompting. Missing required fields are errors. Optional fields default to `undefined`.
- **REPL mode** (`/add-task`): Missing required fields trigger text prompts. Fields in `promptOptional` trigger prompts. Enum fields show selection menus.

### 4. Test the prompting behavior

In the REPL:
1. Type `/add-task` with no arguments — required fields should be prompted
2. Type `/add-task --title "test"` — only `promptOptional` fields should be prompted
3. Enum fields should show a selection list, not free text input
