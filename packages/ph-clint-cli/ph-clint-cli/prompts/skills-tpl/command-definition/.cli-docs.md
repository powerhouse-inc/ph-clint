## API Reference: Command Definition

### `defineCommand<TInput, TOutput, TConfig>(options)`

Identity function that returns the options object with full type inference. Accepts three generic parameters:

- `TInput extends z.ZodType` ŌĆö the input schema type
- `TOutput` ŌĆö the return type of `execute`
- `TConfig` ŌĆö the config type from `defineCli`

### `Command<TInput, TOutput, TConfig>` Interface

```typescript
interface Command<TInput, TOutput, TConfig> {
  id: string;                    // Kebab-case identifier (e.g. 'add-task')
  description: string;           // One-line description for help text
  inputSchema: TInput;           // Zod object schema ŌĆö each field = CLI flag
  outputSchema?: z.ZodType;      // Optional output validation schema
  prompt?: PromptConfig;         // Interactive prompting configuration
  execute: (
    input: z.output<TInput>,
    context: CommandContext<TConfig>
  ) => Promise<TOutput>;
}
```

### `CommandContext<TConfig>` Interface

Available properties on the context object:

| Property | Type | Description |
|----------|------|-------------|
| `workdir` | `string` | Resolved working directory |
| `workspace` | `WorkdirStore` | Persistent file-based state (`.ph/{cliName}/`) |
| `config` | `TConfig` | Typed configuration object |
| `stdout` | `(text: string) => void` | Output function for streaming text |
| `log` | `Logger?` | Structured logger (debug/info/warn/error) |
| `routine` | `Routine?` | Routine loop control (if triggers configured) |
| `processes` | `ProcessManager?` | Run shell commands with timeout and labeling |
| `services` | `ServiceManager?` | Manage background services (if configured) |
| `emit` | `EmitFn?` | Event bus emitter |
| `reactor` | `() => Promise<ReactorContext?>` | Lazy Powerhouse reactor access |
| `agent` | `() => Promise<AgentProvider?>` | Lazy Mastra agent access |

### `WorkdirStore` Methods

```typescript
workspace.loadJsonObject<T>(filename, fallback)   // Load JSON from .ph/ store
workspace.storeJsonObject(filename, value)         // Save JSON to .ph/ store
workspace.loadLocalConfig()                        // Load local config
workspace.storeLocalConfig(config)                 // Save local config
workspace.getWorkdir()                             // Get workdir path
workspace.getStoreFolder(subpath?)                 // Get .ph/ store folder path
```

### `PromptConfig` Interface

```typescript
interface PromptConfig {
  promptForDefaults?: boolean;   // Prompt for fields with .default()? (default: true)
  promptOptional?: string[];     // List of optional field IDs to prompt for
}
```

### Return Value Handling

The framework processes return values as follows:

1. `undefined` or `null` ŌĆö nothing is printed
2. `string` ŌĆö displayed directly as output
3. Object with `.text` property ŌĆö the `text` string is displayed; `.data` is available for programmatic/agent consumption
4. Any other value ŌĆö converted via `String(value)`

### Registration in `defineCli`

Commands are registered via the `commands` array:

```typescript
const cli = defineCli({
  name: 'mycli',
  commands: [greet, addTask, listTasks],
  // ...
});
```
