# Phase 0: Assess Requirements

Before writing any code, clarify what the command should do.

## Steps

### 1. Name the command

Choose a kebab-case, verb-noun name that describes the action:
- `add-task`, `list-images`, `run-build`, `show-status`
- The `id` becomes the CLI subcommand and REPL `/command` name

### 2. Identify all inputs

For each parameter, determine:
- **Type**: `z.string()`, `z.number()`, `z.boolean()`, `z.enum([...])`, `z.array(z.string())`
- **Required vs optional**: Is this mandatory or can it be omitted?
- **Default value**: What's a sensible default? Use `.default(value)` instead of `.optional()` when there is one
- **Validation**: Regex patterns, min/max ranges, URL format, etc.
- **Description**: One-line user-facing text for `--help`

### 3. Determine context needs

Which `CommandContext` properties does the command need?

- [ ] `workspace` — persistent state across invocations
- [ ] `config` — typed CLI configuration
- [ ] `processes` — spawn shell commands
- [ ] `services` — manage background services
- [ ] `stdout` — streaming output (for long-running operations)
- [ ] `log` — debug/info/warn/error logging
- [ ] `reactor()` — Powerhouse document operations
- [ ] `agent()` — AI agent access

### 4. Determine return shape

- **Display-only**: return a string
- **Agent-compatible**: return `{ text: 'human message', data: structuredResult }`
- **Silent**: return `undefined` (rare — most commands should confirm what they did)

### 5. Determine prompting needs

If the command will be used in interactive/REPL mode:
- Which optional fields should be prompted for?
- Should fields with defaults be re-prompted? (Usually no — set `promptForDefaults: false`)
- Enum fields get selection prompts automatically

### 6. Detect codegen project

Check if `cli.ts` contains `@clint:begin` markers. If it does, this is a codegen-managed project:
- Import `defineCommand` from `../framework.js`, not from `@powerhousedao/ph-clint`
- Do NOT manually edit marker regions in `cli.ts` — update `project-spec.json` instead and run `{{commands.clint-project-regen.id}}` to regenerate

### 7. Check for existing commands

- Are there existing commands that overlap with this one?
- Does this command depend on another command's output?
- Could this be an extension of an existing command instead?
