# Phase 1: Define the Input Schema

Design and write the Zod `inputSchema` that defines the command's parameters.

## Steps

### 1. Choose field types

Select the appropriate Zod type for each parameter:

```typescript
z.string()                         // Free text
z.string().url()                   // URL validation
z.string().regex(/^[a-z-]+$/)      // Pattern validation
z.number()                         // Numeric input
z.number().min(1).max(100)         // Bounded number
z.boolean()                        // Flag (--verbose, --dry-run)
z.enum(['low', 'medium', 'high'])  // Fixed choices (auto-completion in REPL)
z.array(z.string())                // Multiple values (--tag a --tag b)
```

### 2. Add `.describe()` to every field

This is mandatory. The description text appears in help output, agent tool descriptions, and prompting labels:

```typescript
z.string().describe('Name of the project to initialize')
z.boolean().default(false).describe('Run without making changes')
```

### 3. Add `.default()` for sensible defaults

Fields with defaults become optional CLI flags. Without a default, the flag is mandatory:

```typescript
z.number().default(3000).describe('Port number')          // --port is optional, defaults to 3000
z.enum(['json', 'table']).default('table').describe('Output format')
```

### 4. Mark truly optional fields

Use `.optional()` only when there is genuinely no default value and the field can be absent:

```typescript
z.string().optional().describe('Due date (YYYY-MM-DD)')   // --due can be omitted entirely
```

### 5. Add validation constraints

```typescript
z.string().regex(/^[a-zA-Z0-9-]+$/).describe('Project name (alphanumeric + hyphens)')
z.number().min(1).max(65535).describe('Port number')
z.string().url().describe('API endpoint URL')
```

### 6. Assemble the schema

```typescript
const inputSchema = z.object({
  name: z.string().describe('Project name'),
  port: z.number().default(3000).describe('Dev server port'),
  template: z.enum(['minimal', 'full']).default('minimal').describe('Project template'),
  dryRun: z.boolean().default(false).describe('Preview changes without writing files'),
});
```

### 7. Verify

- Every required field without a default will be a mandatory `--flag`
- Every field has `.describe()`
- Field names are camelCase (they become `--camel-case` CLI flags automatically)
- Enum types list all valid options
