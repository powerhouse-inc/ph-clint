# Phase 2: Implement the Execute Function

Write the command's logic inside the `execute` function.

## Steps

### 1. Destructure input and context

Only destructure what you need:

```typescript
execute: async ({ name, port }, { workspace, config }) => {
  // `name` and `port` are validated by Zod
  // `workspace` and `config` are injected by the framework
}
```

### 2. Implement the command logic

The body is a regular async function. Use the context resources:

**Workspace (persistent state)**:
```typescript
const tasks = await workspace.loadJsonObject<Task[]>('tasks.json', []);
tasks.push(newTask);
await workspace.storeJsonObject('tasks.json', tasks);
```

**Processes (shell commands)**:
```typescript
const result = await processes!.run('npm install', {
  label: 'install',
  timeout: 60_000,
});
```

**Streaming output (long operations)**:
```typescript
context.stdout('Building project...\n');
// ... work ...
context.stdout('Done.\n');
```

### 3. Choose the return shape

**Simple display** — return a string:
```typescript
return `Created project "${name}" at port ${port}`;
```

**Agent-compatible** — return `{ text, data }`:
```typescript
const project = { name, port, path: projectDir };
return {
  text: `Created project "${name}" at ${projectDir}`,
  data: project,
};
```

### 4. Handle errors

Throw errors — the framework catches them and formats the output:

```typescript
if (!fs.existsSync(targetDir)) {
  throw new Error(`Directory not found: ${targetDir}`);
}
```

Do NOT:
- Return error strings (the exit code will be 0)
- Call `process.exit(1)` (breaks testability)
- Use `console.error()` (not captured by RunOptions)

### 5. Assemble the full command

```typescript
import { defineCommand } from '../framework.js';
import { z } from 'zod';

export const myCommand = defineCommand({
  id: 'my-command',
  description: 'Does something useful',
  inputSchema: z.object({
    name: z.string().describe('The name'),
    verbose: z.boolean().default(false).describe('Show detailed output'),
  }),
  execute: async ({ name, verbose }, { workspace, config }) => {
    // Implementation here
    return { text: `Done: ${name}`, data: { name } };
  },
});
```

### 6. Export and register

Export the command from its file, then register it in the CLI.

**In a codegen-managed project** (cli.ts has `@clint:begin` markers):

Do NOT manually edit `cli.ts`. Instead:

1. Add the command to `project-spec.json`
2. Run `{{commands.clint-project-regen.id}}` — this regenerates the marker regions with the correct import and registration
3. Run `{{commands.clint-project-build.id}}` to verify TypeScript compiles

**In a manually-managed project** (no markers):

```typescript
import { myCommand } from './commands/my-command.js';

const cli = defineCli({
  commands: [...existingCommands, myCommand],
  // ...
});
```

In both cases, import `defineCommand` from `../framework.js` in your command file (not from `@powerhousedao/ph-clint`) to get automatic config typing.
