## API Reference: Service Definition

### `defineService<TConfig>(options): ServiceDefinition<TConfig>`

Identity function that returns the options object with full type inference. One generic parameter:

- `TConfig` ‚Äî the config type from `defineCli` (for typed `env()` function)

### `ServiceDefinition<TConfig>` Interface

```typescript
interface ServiceDefinition<TConfig = Record<string, unknown>> {
  id: string;                          // Kebab-case identifier (e.g. 'dev-server')
  command: string | ((params?: Record<string, unknown>) => string);
  name?: string;                       // Display name (e.g. 'Dev Server')
  description?: string;                // One-line description
  env?: (config: TConfig, params?: Record<string, unknown>) => Record<string, string>;
  paramsSchema?: ZodType;              // Runtime parameters (become start command flags)
  maxInstances?: number;               // Concurrency limit (default: 1)
  preflight?: PreflightCheck<TConfig>[];
  readiness?: ReadinessConfig;
  shutdown?: { signal: NodeJS.Signals; timeout: number };
  restart?: { enabled: boolean; maxRetries: number; delay: number };
  projectScanner?: ProjectScanner;
}
```

### `ReadinessConfig` ‚Äî Single Pattern Form

```typescript
{
  pattern: RegExp;
  captures?: Record<string, number | CaptureDefinition>;
  timeout: number;       // ms before marking failed
  wait?: boolean;        // true (default) = block until match; false = ready immediately
}
```

### `ReadinessConfig` ‚Äî Multiple Patterns Form

```typescript
{
  patterns: ReadinessPattern[];
  timeout: number;
  wait?: boolean;
}
```

ALL patterns must match before the service is considered ready.

### `ReadinessPattern`

```typescript
interface ReadinessPattern {
  name: string;           // Identifier for progress reporting
  pattern: RegExp;        // Regex to match against stdout/stderr lines
  captures?: Record<string, number | CaptureDefinition>;
}
```

### `CaptureDefinition`

```typescript
interface CaptureDefinition {
  group: number;          // 1-indexed regex capture group
  type?: EndpointType;    // Enables downstream features (e.g. MCP discovery)
}
```

### `EndpointType` Union

```typescript
type EndpointType = 'other' | 'api-mcp' | 'api-rest' | 'api-graphql' | 'website';
```

### Preflight Checks

```typescript
type PreflightCheck<TConfig> =
  (ctx: PreflightContext<TConfig>) => PreflightResult | Promise<PreflightResult>;

interface PreflightContext<TConfig> {
  cwd: string;
  config: TConfig;
  params?: Record<string, unknown>;
  command: string;             // Resolved command string
}

type PreflightResult =
  | { ok: true }
  | { ok: false; message: string; hint?: string };
```

### Built-in Preflight Factories

**`checkWorkdir(test, message, hint?)`**
```typescript
checkWorkdir(
  test: (cwd: string) => boolean,   // Return true if valid
  message: string,                   // Error message on failure
  hint?: string                      // Suggested fix
): PreflightCheck
```

**`checkCommand(binary, options?)`**
```typescript
checkCommand(
  binary: string,
  options?: {
    versionFlag?: string;                          // Default: '--version'
    versionTest?: (version: string) => boolean;    // Validate version output
    hint?: string;                                 // Install instructions
  }
): PreflightCheck
```

**`checkPort(port, label?)`**
```typescript
checkPort(
  port: number | ((ctx: PreflightContext) => number | undefined),
  label?: string               // Display name (e.g. 'Dev Server')
): PreflightCheck
```

**`isPortFree(port): Promise<boolean>`** ‚Äî utility for custom checks.

### `ServiceManager` Interface

Methods available on `context.services`:

| Method | Signature | Description |
|--------|-----------|-------------|
| `start` | `(id, opts?: ServiceStartOptions) => Promise<string>` | Start service, returns instanceId |
| `stop` | `(id, instanceId?) => Promise<void>` | Stop service instance |
| `list` | `(serviceId?) => ServiceInstanceStatus[]` | List instance statuses |
| `getDefinition` | `(id) => ServiceDefinition \| undefined` | Look up a service definition |
| `logs` | `(id, instanceId?, lines?) => string` | Tail service log (default 50 lines) |
| `watchLogs` | `(id, instanceId, onLine) => () => void` | Stream log lines, returns unsubscribe |
| `scanProjects` | `(id, rootDir) => ProjectScanResult[]` | Discover projects for a service |
| `purgeStoppedInstances` | `(id) => void` | Clean up stopped instance state |

### `ServiceStartOptions`

```typescript
interface ServiceStartOptions {
  workdir?: string;                    // For instance identity (hashed into instanceId)
  cwd?: string;                        // Spawn directory (defaults to process.cwd())
  params?: Record<string, unknown>;    // Runtime parameters
  name?: string;                       // Named instance (for multi-instance services)
}
```

### `ServiceInstanceStatus`

```typescript
interface ServiceInstanceStatus {
  serviceId: string;
  instanceId: string;
  name: string;
  status: 'idle' | 'starting' | 'ready' | 'failed' | 'stopping' | 'stopped';
  pid?: number;
  endpoints?: Record<string, string>;
  endpointTypes?: Record<string, EndpointType>;
  error?: string;
  restartAttempt?: number;
  workdir?: string;
  params?: Record<string, unknown>;
}
```

### Service Events

| Event | Payload | When |
|-------|---------|------|
| `service:ready` | `{ id, instanceId, name, endpoints?, endpointTypes? }` | All readiness patterns matched |
| `service:failed` | `{ id, instanceId, name, error }` | Readiness timeout or process crash |
| `service:stopped` | `{ id, instanceId, name }` | Explicit stop completed |
| `service:pattern-matched` | `{ id, instanceId, name, patternName, endpoints, endpointTypes, remaining }` | One readiness pattern matched |
| `service:restarting` | `{ id, instanceId, name, attempt, maxRetries }` | Auto-restart triggered |

### `ProjectScanner` Interface

```typescript
interface ProjectScanner {
  isProjectFolder(folderPath: string): boolean;
  getProjectName?(folderPath: string): string;
  getProjectConfig?(folderPath: string): Record<string, unknown>;
  getDocumentLink?(folderPath: string): { documentId: string; documentType: string } | undefined;
}
```

### Auto-Generated Commands per Service

| Command ID | Description |
|-----------|-------------|
| `{id}-start` | Start the service (flags from paramsSchema + workdir + name) |
| `{id}-stop` | Stop a running instance |
| `{id}-restart` | Restart preserving workdir/params |
| `{id}-ps` | Show instance status |
| `{id}-logs` | Tail service logs (default 50 lines) |
| `{id}-manage` | Interactive management panel (REPL only) |
| `{id}-ls` | List discovered projects (only if projectScanner defined) |

### Registration in `defineCli`

Services are registered via the `services` array:

```typescript
const cli = defineCli({
  name: 'mycli',
  services: [devServer, workerPool],
  // ...
});
```
