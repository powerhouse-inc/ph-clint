# Phase 0: Assess Service Requirements

Before writing any code, identify what the service needs.

## Steps

### 1. Identify the process

What does this service run?

- **Static command**: `'pnpm dev'` — always the same
- **Parameterized command**: `(params) => \`pnpm dev -p ${params?.port ?? 3000}\`` — varies per invocation

### 2. Determine readiness signals

What does the process print to stdout/stderr when it's ready?

- Copy the **exact line(s)** from the process output
- Identify the regex that matches each line
- Note any URLs, ports, or other values to capture from those lines
- If the process prints multiple readiness signals (e.g., Connect Studio URL + Switchboard URL + MCP URL), each needs its own pattern — all must match

### 3. Identify endpoints

What endpoints does the service expose?

- [ ] HTTP REST API → `type: 'api-rest'`
- [ ] GraphQL API → `type: 'api-graphql'`
- [ ] MCP server → `type: 'api-mcp'` (enables agent tool discovery)
- [ ] Website/UI → `type: 'website'`
- [ ] Other → `type: 'other'`

For each endpoint, note which stdout line contains its URL and which regex group captures it.

### 4. Identify preflight requirements

What must be true before the process can start?

- [ ] **Working directory**: Does it need specific files? (`package.json`, `powerhouse.config.ts`, etc.)
- [ ] **Binaries**: Does it need installed tools? (`node`, `pnpm`, `ph`, `docker`, etc.)
- [ ] **Ports**: What ports does it bind to? Are they configurable?

### 5. Determine lifecycle policies

- **Shutdown**: Which signal does the process respond to? (Most Node.js processes: SIGTERM)
- **Restart on crash?** How many retries? What delay between attempts?
- **Multiple instances?** Can several run simultaneously, or is it exclusive? (`maxInstances`)

### 6. Determine runtime parameters

Does the service need per-invocation parameters?

- Ports, URLs, feature flags → define `paramsSchema` (Zod schema)
- Parameters become flags on the auto-generated `{id}-start` command
- Parameters are passed to both `command()` and `env()` functions

### 7. Determine environment variables

Does the process need environment variables derived from CLI config or params?

- `env(config, params) => Record<string, string>`
- Common: `PORT`, `NODE_ENV`, `HOST`, API URLs, feature flags

### 8. Check for project scanning needs

Does one service definition serve multiple project directories?

- If yes → define `projectScanner` to enable `{id}-ls` command
- Scanner needs `isProjectFolder(path)` predicate at minimum
