# Phase 2: Add Preflight Validation Checks

Add checks that run before the service process spawns. These catch common failure modes early with clear error messages.

## Steps

### 1. Add workdir check

Use `checkWorkdir()` when the service requires specific directory structure:

```typescript
import { checkWorkdir } from '../framework.js';
import fs from 'node:fs';
import path from 'node:path';

preflight: [
  checkWorkdir(
    (cwd) => fs.existsSync(path.join(cwd, 'package.json')),
    'Not a Node.js project',
    'Run {id}-start --workdir <project-dir>, or create a project first',
  ),
],
```

The `test` function receives the resolved working directory. Return `true` if valid.

You can check for multiple files:
```typescript
checkWorkdir(
  (cwd) => fs.existsSync(path.join(cwd, 'powerhouse.config.ts'))
         || fs.existsSync(path.join(cwd, 'powerhouse.config.json')),
  'Not a Reactor Package project',
  'Run {id}-start --workdir <project>, or create one with /init',
),
```

### 2. Add binary checks

Use `checkCommand()` for required binaries:

```typescript
import { checkCommand } from '../framework.js';

checkCommand('node', {
  hint: 'Install Node.js from https://nodejs.org',
}),
checkCommand('pnpm', {
  hint: 'Install pnpm: npm install -g pnpm',
}),
checkCommand('ph', {
  hint: 'Install the Powerhouse CLI: npm install -g ph-cli',
}),
```

With version validation:
```typescript
checkCommand('node', {
  versionFlag: '--version',
  versionTest: (v) => parseInt(v.replace('v', '')) >= 22,
  hint: 'Node.js >= 22 is required. Update from https://nodejs.org',
}),
```

### 3. Add port checks

Use `checkPort()` for each port the service binds to:

**Static port**:
```typescript
import { checkPort } from '../framework.js';

checkPort(3000, 'Dev Server'),
```

**Dynamic port from params**:
```typescript
checkPort((ctx) => (ctx.params?.port as number) ?? 3000, 'Dev Server'),
```

**Multiple ports**:
```typescript
checkPort((ctx) => (ctx.params?.connectPort as number) ?? 3000, 'Connect Studio'),
checkPort((ctx) => (ctx.params?.switchboardPort as number) ?? 4001, 'Switchboard'),
```

### 4. Order checks correctly

Place checks in order from cheapest to most expensive:

```typescript
preflight: [
  // 1. Workdir check — pure filesystem, instant
  checkWorkdir(
    (cwd) => fs.existsSync(path.join(cwd, 'package.json')),
    'Not a Node.js project',
  ),
  // 2. Binary check — spawns a process, fast
  checkCommand('pnpm', { hint: 'Install pnpm: npm install -g pnpm' }),
  // 3. Port check — network I/O, slowest
  checkPort((ctx) => (ctx.params?.port as number) ?? 3000, 'Dev Server'),
],
```

### 5. Write custom preflight checks (if needed)

For checks not covered by the built-in factories, write a custom function:

```typescript
preflight: [
  async (ctx) => {
    const configPath = path.join(ctx.cwd, '.env');
    if (!fs.existsSync(configPath)) {
      return {
        ok: false,
        message: 'Missing .env file',
        hint: 'Copy .env.example to .env and fill in the values',
      };
    }
    return { ok: true };
  },
],
```

Custom checks receive `PreflightContext` with `cwd`, `config`, `params`, and `command`.

### 6. Write clear error messages and hints

Every failing check should tell the user:
- **What's wrong**: `'Port 3000 is already in use'`
- **How to fix it** (hint): `'Stop the other process or use --port <number>'`

Good hints save users from having to search for solutions.
