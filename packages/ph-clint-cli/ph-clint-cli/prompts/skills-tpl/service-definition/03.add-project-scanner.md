# Phase 3: Add Project Scanner (Optional)

Add project scanning when one service definition can manage multiple project directories. Skip this phase if the service always runs in a single fixed location.

## When to Add a Scanner

- A dev server that can run in any project folder matching a pattern
- A reactor service that works with any Powerhouse project
- Any service where users need to discover eligible project directories

## Steps

### 1. Define the scanner

At minimum, implement `isProjectFolder`:

```typescript
import fs from 'node:fs';
import path from 'node:path';

export const myService = defineService({
  // ...
  projectScanner: {
    isProjectFolder: (p) => fs.existsSync(path.join(p, 'next.config.ts')),
  },
});
```

### 2. Add getProjectName (recommended)

Extract a display name from the project directory:

```typescript
projectScanner: {
  isProjectFolder: (p) => fs.existsSync(path.join(p, 'package.json')),
  getProjectName: (p) => {
    const pkg = JSON.parse(fs.readFileSync(path.join(p, 'package.json'), 'utf-8'));
    return pkg.name ?? path.basename(p);
  },
},
```

### 3. Add getProjectConfig (optional)

Return project-specific configuration:

```typescript
getProjectConfig: (p) => {
  const pkg = JSON.parse(fs.readFileSync(path.join(p, 'package.json'), 'utf-8'));
  return { name: pkg.name, version: pkg.version };
},
```

### 4. Add getDocumentLink (optional, Powerhouse projects)

Return a document reference for Powerhouse-linked projects:

```typescript
getDocumentLink: (p) => {
  const configPath = path.join(p, 'powerhouse.config.json');
  if (!fs.existsSync(configPath)) return undefined;
  const config = JSON.parse(fs.readFileSync(configPath, 'utf-8'));
  return config.documentId
    ? { documentId: config.documentId, documentType: config.documentType }
    : undefined;
},
```

### 5. How scanning works

When `projectScanner` is defined:
- The auto-generated `{id}-ls` command becomes available
- `services.scanProjects(serviceId, rootDir)` performs a breadth-first search
- The scanner prunes subtrees at foreign project boundaries (`package.json`, `Cargo.toml`, etc.)
- Returns `ProjectScanResult[]` with `{ name, path, config?, documentId?, documentType? }`

### 6. Using scan results

Scan results are useful for:
- Listing projects in the REPL (`/{id}-ls`)
- Auto-starting services for discovered projects
- Building project selection menus in interactive mode
