## Playwright CLI

Automate browser interactions through the `playwright-cli` command-line tool. Use it to navigate web pages, interact with elements, take screenshots, manage sessions, run tests, and extract information from running applications — all from within the agent conversation.

## When to use

Use this skill when you need to visually verify a running application, test UI interactions, fill out forms, capture screenshots, debug frontend behavior, or automate any browser-based workflow. Particularly useful after deploying a Fusion page or editor to verify it renders and behaves correctly. Typical prompts: "Navigate to the dashboard and take a screenshot" or "Run the Playwright test suite."

## Parameters

- **prompt** (required): The browser automation task to perform.
- **useBrowser** (default: `chromium`): Browser engine to use for the session.
  - `chromium` — Playwright's bundled Chromium (default, most compatible)
  - `chrome` — System-installed Google Chrome
  - `firefox` — Mozilla Firefox
  - `webkit` — WebKit (Safari's engine)
  - `msedge` — Microsoft Edge

## Example invocations

```
/playwright-cli --prompt "Navigate to http://localhost:3000 and take a screenshot of the dashboard"
/playwright-cli --useBrowser firefox --prompt "Test the login form with invalid credentials and capture the error message"
/playwright-cli --prompt "Run the Playwright test suite and report failures"
```

## Workflow

The agent uses `playwright-cli` commands to automate browser sessions:

### Session lifecycle
Opens a browser session with `playwright-cli open [url]`, passing the configured `--browser` flag. Interacts with the page through snapshot-based element targeting (refs like `e15`), CSS selectors, or Playwright locators. Closes the session when done.

### Core interactions
Navigates with `goto`, `go-back`, `go-forward`, `reload`. Interacts with elements via `click`, `fill`, `type`, `select`, `hover`, `drag`, `check/uncheck`. Takes snapshots to inspect current page state and identify element refs. Evaluates JavaScript with `eval` for custom queries.

### Testing and debugging
Runs Playwright test suites, captures console output and network activity, records traces with `tracing-start`/`tracing-stop`, and records video with `video-start`/`video-stop`. Manages request mocking with `route`/`unroute` for testing against simulated backends.

### Multi-tab and session management
Supports multiple tabs (`tab-new`, `tab-select`, `tab-close`) and named sessions (`-s=mysession`) for parallel browser workflows. Manages storage state, cookies, localStorage, and sessionStorage for authentication and state persistence.

### Output handling
Uses the `--raw` flag to strip page status and snapshot sections from output, returning only the result value for piping into other tools or capturing structured data.
