## Document Editor Creation

Build React editor components that provide the user interface for viewing and editing Powerhouse document types. Editors are registered through the Vetra MCP, which generates the scaffold and hooks; this skill guides the agent through registration, implementation, and stakeholder UAT.

## When to use

Use this skill when you need to create a new editor for an existing document model, redesign an editor's layout, or add viewing/editing capabilities. The document model should already exist before starting editor creation. Typical prompts: "Create a form-based editor for the invoice document model" or "Add inline line-item editing to the invoice editor."

## Parameters

- **prompt** (required): Description of the editor to build or modify.
- **mode** (default: `expert`):
  - `expert` — The agent confirms layout decisions, component structure, and interaction patterns with you before implementing. Best for developers who want control over UI/UX choices.
  - `discovery` — The agent explains the editor architecture (hooks, dispatch, creators) as it builds, and walks you through each decision. Best for users unfamiliar with the Powerhouse editor pattern.
  - `one-shot` — The agent makes all design decisions autonomously and implements the full editor without asking. Best when you trust the defaults or have a detailed specification.
- **useBrowser** (default: `chromium`): Browser engine for Playwright visual verification of the editor. Options: `chromium`, `chrome`, `firefox`, `webkit`, `msedge`.

## Example invocations

```
/document-editor-creation --prompt "Create an editor for the invoice document with inline line-item editing"
/document-editor-creation --mode discovery --prompt "Build a simple viewer for the contact list document model"
/document-editor-creation --mode one-shot --useBrowser firefox --prompt "Create a task board editor with drag-and-drop status columns"
```

## Workflow

The agent follows a structured two-phase process:

### Phase 1: Check prerequisites
Verifies the Reactor Package project is running via `{{commands.reactor-project-ps.id}}` and `{{commands.reactor-project-logs.id}}`. Familiarizes with the target document model by reading its state schema and operations through the `{{services.reactor-project.mcpPrefix}}*` tools. Checks whether an editor already exists (and if so, asks whether to create a new one or modify the existing one). Creates the editor specification via `{{services.reactor-project.mcpPrefix}}createDocument` with type `powerhouse/document-editor`, then configures it with `{{services.reactor-project.mcpPrefix}}addActions` using `ADD_DOCUMENT_TYPE`, `SET_EDITOR_NAME`, and `SET_EDITOR_STATUS` (CONFIRMED). Waits for the code generator to produce the editor scaffold in `editors/` (confirmed via `{{commands.reactor-project-logs.id}}`) and inspects the generated hooks.

### Phase 2: Write editor implementation
Cleans up generated boilerplate and implements the editor UI using the modern React hooks pattern from `@powerhousedao/reactor-browser`. The implementation uses generated hooks like `useSelectedXDocument()` for document state and `dispatch()` with action creators for mutations. The agent creates modular components, styles with Tailwind or scoped style tags, creates test documents on the Preview Drive via `{{services.reactor-project.mcpPrefix}}*` tools to verify functionality, implements both viewing and editing features, and runs type/lint checks. After resolving all issues, uses the `playwright-cli` skill with the configured browser to visually verify the editor in Vetra Studio. Concludes by sending a UAT invitation to stakeholders.

## Key patterns

- Editors use `useSelectedXDocument()` hooks (auto-generated) for state access
- Mutations dispatch action creators from `document-models/*/gen/creators.js`
- New IDs are generated client-side with `generateId()` from `document-model/core`
- Never edit files in `gen/` folders — they are overwritten on regeneration
- Consider using components from `@powerhousedao/design-system` and `@powerhousedao/document-engineering`
