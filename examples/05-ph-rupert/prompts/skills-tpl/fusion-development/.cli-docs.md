## Fusion Development

Build pages and components in a Fusion application — a Next.js platform that uses Powerhouse document drives as its backend via a Switchboard GraphQL API. This skill covers the full page implementation workflow from routing to GraphQL integration to rendered UI.

## When to use

Use this skill when you need to create or modify pages in a Fusion (Next.js) project that displays or interacts with data from Powerhouse document drives. The Reactor Package backend and Fusion project should already be running. Typical prompts: "Create a dashboard page listing all invoice documents" or "Add a document detail view with status filters."

## Parameters

- **prompt** (required): Description of the page or feature to build.
- **mode** (default: `expert`):
  - `expert` — The agent discusses routing structure, GraphQL query design, and component architecture with you before implementing. Best for developers who want to guide the technical approach.
  - `discovery` — The agent explains the Fusion architecture (Next.js app router, GraphQL codegen, server vs client components) as it works, helping you understand how each piece fits together. Best for users new to Fusion or Next.js.
  - `one-shot` — The agent makes all implementation decisions autonomously and builds the full page without asking. Best when you have a clear specification or want a quick prototype.

## Example invocations

```
/fusion-development --prompt "Create a dashboard page listing all invoice documents with status filters"
/fusion-development --mode discovery --prompt "Build a page that shows document drive contents in a tree view"
/fusion-development --mode one-shot --prompt "Add a detail page at /invoices/[id] showing all invoice fields"
```

## Workflow

The agent follows a structured four-step process:

### Step 1: Ensure services are running
Verifies that the required Reactor Package backend is running (via `{{commands.reactor-project-ls.id}}` / `{{commands.reactor-project-start.id}}`) and the Fusion project is running (via `{{commands.fusion-project-ls.id}}` / `{{commands.fusion-project-start.id}}`). Familiarizes with the existing Fusion codebase.

### Step 2: Set up routes and GraphQL queries
Creates the Next.js route structure (`/app/...` folders with `page.tsx` and `layout.tsx` files). Inspects the Switchboard GraphQL backend, identifies or writes the required `.graphql` query files in `/modules/**/graphql/`, runs `pnpm codegen` to generate typed client helpers, and sets up query functions in the page (server-side fetcher functions or client-side hooks depending on the component type).

### Step 3: Implement page components
Builds the React components to display the fetched data. Starts with read-only rendering, verifies with `pnpm build`, and iterates on the UI. Server components use async fetcher functions; client components use generated query hooks directly.

### Step 4: Stakeholder UAT
Sends a message to stakeholders through the inbox inviting them to test the new page.
