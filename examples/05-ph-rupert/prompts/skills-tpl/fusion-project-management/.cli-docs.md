## Fusion Project Management

Initialize, configure, and manage the lifecycle of Fusion project instances — Next.js applications that use Powerhouse document drives as their backend. This skill handles project creation, Switchboard backend configuration, dev server startup, and shutdown.

## When to use

Use this skill when you need to create a new Fusion project, start or stop a Fusion dev server, or configure the Switchboard URL that the project connects to. This is typically the first step before doing any Fusion development work. Typical prompts: "Initialize a new Fusion project" or "Start the Fusion dev server for my-platform."

## Parameters

- **prompt** (required): The management task to perform.
- **mode** (default: `expert`):
  - `expert` — The agent confirms project naming and backend configuration choices with you before proceeding. Best when you need control over which Switchboard instance to connect to.
  - `discovery` — The agent explains the Fusion project structure, what a Switchboard is, and why backend configuration matters as it works. Best for users new to the Powerhouse platform.
  - `one-shot` — The agent makes all configuration decisions autonomously and completes the task without asking. Best for routine project management operations.

## Example invocations

```
/fusion-project-management --prompt "Initialize a new Fusion project called 'acme-platform'"
/fusion-project-management --mode discovery --prompt "Set up a Fusion project connected to the local Reactor backend"
/fusion-project-management --mode one-shot --prompt "Start the Fusion dev server for acme-platform"
```

## Workflow

The skill covers four operations that can be invoked independently:

### Initialize project (FPM.00)
Lists existing Fusion projects with `{{commands.fusion-project-ls.id}}`, verifies no project is currently running (stops any running project with `{{commands.fusion-project-stop.id}}`), generates a unique kebab-case project name, and creates the project with `{{commands.fusion-project-init.id}}`. Confirms the new project appears in the project list.

### Obtain Switchboard URL (FPM.01)
Determines which Switchboard backend the Fusion project should connect to. Considers whether a cloud-hosted Switchboard URL was provided, or whether a local Reactor Package project needs to be started first. If a local backend is needed, ensures the Reactor Package is running via `{{commands.reactor-project-ls.id}}` and `{{commands.reactor-project-start.id}}`. Verifies the Switchboard URL format (`http(s)://host[:port]/graphql`).

### Run project (FPM.02)
Starts the Fusion dev server with `{{commands.fusion-project-start.id}}`, providing the project name and Switchboard URL. Polls `{{commands.fusion-project-ps.id}}` until the project is ready (up to 90 seconds). Captures the Fusion URL and port from startup logs via `{{commands.fusion-project-logs.id}}`.

### Stop project (FPM.03)
Verifies the project is running with `{{commands.fusion-project-ps.id}}`, stops the Next.js dev server with `{{commands.fusion-project-stop.id}}`, and confirms shutdown.
