## Reactor Project Management

Initialize and manage the lifecycle of Reactor Package projects — the development environment for building Powerhouse document models, editors, and drive apps. This skill handles project creation, Vetra service startup (Connect + Switchboard), endpoint discovery, and shutdown.

## When to use

Use this skill when you need to create a new Reactor Package project, start or stop Vetra services, or check project status. This is typically the first step before any document modeling or editor creation work. Typical prompts: "Create a new Reactor Package project" or "Start the Vetra server for my-docs."

## Parameters

- **prompt** (required): The management task to perform.
- **mode** (default: `expert`):
  - `expert` — The agent confirms project naming and startup decisions with you before proceeding. Best when managing multiple projects or specific configurations.
  - `discovery` — The agent explains what a Reactor Package is, what Vetra Studio and Vetra Switchboard do, and what the MCP tools are for as it works. Best for users new to the Powerhouse development environment.
  - `one-shot` — The agent makes all decisions autonomously and completes the task without asking. Best for routine operations like starting a known project.

## Example invocations

```
/reactor-project-management --prompt "Initialize a new Reactor Package project called 'acme-invoicing'"
/reactor-project-management --mode discovery --prompt "Set up a new project for building a task management system"
/reactor-project-management --mode one-shot --prompt "Start the Vetra server for acme-invoicing"
```

## Workflow

The skill covers three operations that can be invoked independently:

### Initialize project (RPPM.00)
Lists existing projects with `{{commands.reactor-project-ls.id}}`, verifies no project is currently running (stops any running project with `{{commands.reactor-project-stop.id}}`), generates a unique kebab-case project name, and creates the project with `{{commands.reactor-project-init.id}}`. Confirms the new project appears in the project list.

### Run project and capture endpoints (RPPM.01)
Starts the Reactor Package with `{{commands.reactor-project-start.id}}`, which launches `ph vetra --watch` in the background. Polls `{{commands.reactor-project-ps.id}}` every few seconds until both Vetra Studio and Vetra Switchboard are ready (up to 90 seconds). Parses startup logs via `{{commands.reactor-project-logs.id}}` to identify the Vetra Studio URL, Switchboard URL, and MCP endpoint. Once running, the `{{services.reactor-project.mcpPrefix}}*` tools become available for creating document models, editors, and other specifications.

### Stop project (RPPM.02)
Verifies the project is running with `{{commands.reactor-project-ps.id}}`, stops both Vetra Studio and Vetra Switchboard services with `{{commands.reactor-project-stop.id}}`, and confirms the project has stopped.
