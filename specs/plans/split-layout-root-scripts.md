# Plan: Update split-layout root package.json codegen

## Context

The `packages/package.json` in the ph-clint repo itself uses `pnpm --prefix` for orchestrating install/build/test across independent sub-packages. The split-layout root `package.json` generated by `ph-clint-cli` for implementation projects uses `cd && ... && cd ../` chains instead. These should be aligned.

Additionally, implementation projects that opt into publishing need `publish:*` scripts in their root `package.json`, following the same pattern as `packages/package.json`.

## Current state

**`packages/package.json`** (the reference):
- Uses `pnpm --prefix <dir>` for all scripts — cleaner, no cwd mutation
- Has `install`, `build`, `test`, and `publish:*` scripts
- Has `tsx` as a devDependency (needed to run publish-cli from source)

**`root-package-json.ts`** codegen builder (at `packages/ph-clint-cli/ph-clint-cli/src/codegen/builders/root-package-json.ts`):
- Uses `cd {app} && pnpm build && cd ../{cli} && pnpm build` chains
- Has `install`, `build`, `dev`, `start`, `test`, `lint`, `app:dev`, `cli:dev`
- No publish scripts
- No devDependencies

## Changes needed

### 1. Switch to `pnpm --prefix` style

Replace `cd` chains with `pnpm --prefix` in `buildRootPackageJson()`. For a project named `foo`:

```json
{
  "scripts": {
    "install": "pnpm --prefix foo-app install && pnpm --prefix foo-cli install",
    "build": "pnpm --prefix foo-app build && pnpm --prefix foo-cli build",
    "test": "pnpm --prefix foo-app test && pnpm --prefix foo-cli test",
    "dev": "pnpm --prefix foo-cli dev",
    "start": "pnpm --prefix foo-cli start",
    "lint": "pnpm --prefix foo-cli lint",
    "app:dev": "pnpm --prefix foo-app dev",
    "cli:dev": "pnpm --prefix foo-cli dev"
  }
}
```

Build order: app first, then cli (cli depends on app via `file:` dep).

### 2. Add publish scripts (conditional)

When the project has a `publish.config.ts` (check if codegen emits one, or if it should), add:

```json
{
  "scripts": {
    "publish:dev": "tsx foo-cli/node_modules/.bin/ph-publish dev -c ./publish.config.ts",
    "publish:staging": "tsx foo-cli/node_modules/.bin/ph-publish staging -c ./publish.config.ts",
    "publish:production": "tsx foo-cli/node_modules/.bin/ph-publish production -c ./publish.config.ts"
  },
  "devDependencies": {
    "tsx": "^4.19.0"
  }
}
```

Alternatively, since `ph-publish` is a bin provided by `@powerhousedao/ph-clint-dev` (which is a devDependency of the cli sub-package), the root could use `pnpm --prefix foo-cli exec ph-publish dev -c ../publish.config.ts` — avoiding a root-level tsx dependency entirely. Evaluate which approach is cleaner.

### 3. Scaffold `publish.config.ts` for split-layout projects

The codegen should emit a `publish.config.ts` at the project root for split-layout projects. This config would define a single group with both the app and cli packages. Reference `packages/publish.config.ts` for the shape, but note that implementation projects import from the published `@powerhousedao/ph-clint-dev` package, not from source:

```ts
import { definePublishConfig } from '@powerhousedao/ph-clint-dev/publish';

export default definePublishConfig({
  groups: {
    'foo': {
      version: '0.0.1',
      packages: [
        { path: 'foo-app', category: 'app' },
        { path: 'foo-cli', category: 'cli' },
      ],
    },
  },
});
```

### 4. Update tests

Update `tests/codegen/root-package-json.test.ts` to assert `pnpm --prefix` style scripts.
