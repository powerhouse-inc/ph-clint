# DS.00 Define Visual Identity

Prerequisite: Approved messaging (`messaging/approved.md`) exists.

## DS.00.1 Gather brand inputs

- Read `messaging/approved.md` — the tone and audience inform the visual direction
- Read `research/audience.md` — enterprise vs. consumer vs. developer audience
- Check sources for existing brand assets (logo, colors, fonts, style guides)

## DS.00.2 Propose visual directions

<!-- expert mode / discovery mode -->
Present 2-3 visual directions with rationale tied to the audience:
- Color palette (primary, secondary, accent, neutrals, backgrounds, surfaces)
- Typography (heading, body, mono — system fonts or self-hosted)
- Spacing scale (consistent multiplier, e.g., 4px base)
- Border and shadow style (sharp vs. rounded, flat vs. elevated)
- Overall tone (minimal/monochrome, bold/colorful, warm/approachable)

For each direction, explain why it fits the audience. Let the user choose.

<!-- one-shot mode -->
Choose the direction that best matches the audience profile and existing brand assets.

## DS.00.3 Produce identity spec

- Write `design-system/identity.md` with the chosen direction and rationale
- Write `design-system/variables.css` with all CSS custom properties
- Verify contrast ratios meet accessibility minimums (4.5:1 body text, 3:1 large text)

### Expected Scenario Outcome

`design-system/identity.md` and `design-system/variables.css` exist with a complete, accessible color and typography system.
