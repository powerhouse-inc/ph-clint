# DS.01 Build Component Library

Prerequisite: Visual identity defined (DS.00).

## DS.01.1 Build core components

Create each component as HTML + CSS using the design system variables:

1. **Navigation** — Header with logo area, links, primary CTA button. Collapses to hamburger on mobile.
2. **Hero** — Large headline, subheadline, CTA button, optional image/diagram area.
3. **Content section** — Section heading, body text, optional aside or illustration area.
4. **Feature grid** — 2-3 column responsive grid of feature cards with icon, title, description.
5. **CTA banner** — Mid-page conversion block with headline, description, button. Visually distinct.
6. **Comparison table** — Responsive table that collapses to card stack on mobile.
7. **Footer** — Links, contact info, legal text. Full-width.
8. **Code block** — Monospace text with background, optional line numbers.
9. **Quote/testimonial** — Attribution, quote text, optional image.
10. **Icon set** — Inline SVGs for common symbols. Embedded, NOT external.

## DS.01.2 Write component CSS

- All styles in `design-system/components.css`
- Use CSS custom properties from `variables.css` — no hardcoded colors/sizes
- Include responsive breakpoints: 320px, 768px, 1440px
- Include hover/focus/active states for interactive elements

## DS.01.3 Create preview page

- Write `design-system/preview.html` showing all components with example content
- Include `variables.css` and `components.css`
- Organize by component type with headings
- Must be viewable with `npx serve design-system/`

## DS.01.4 Verify

- Open `preview.html` and verify all components render correctly
- Check responsive behavior at 320px, 768px, 1440px
- Verify no external resource references (fonts, icons, CDNs)
- Verify all interactive elements are keyboard-accessible

### Expected Scenario Outcome

`design-system/` contains a complete, self-contained component library. The site prototype will compose these components.
