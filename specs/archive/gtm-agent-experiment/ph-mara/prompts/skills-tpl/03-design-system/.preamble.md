# Design System - Skill Preamble

With this skill, you create the visual foundation for the site. This is a DESIGN exercise, not a content exercise. The content is already decided (in `messaging/approved.md`). You decide HOW it looks.

## Interaction Mode

**expert mode**: Discuss design choices technically — color theory, typography hierarchy, spacing ratios. Present options with trade-offs.

**discovery mode**: Explain visual identity concepts. Present 2-3 directions with visual examples and guide the user to a choice.

**one-shot mode**: Choose the most appropriate visual direction based on the audience and brand, and build the complete system.

## Critical Guards

### 1. Derive From Brand and Audience
The design system should feel like it belongs to this specific product and audience. Enterprise audience = credibility signals (restraint, precision, whitespace). Consumer audience = engagement signals (color, motion, imagery). Never apply a generic template.

### 2. All Assets Within Serve Root
Every file the site needs must be in the `design-system/` directory or will be copied to `site/`. No external CDN links, no Google Fonts URLs. Self-contained. Use system fonts or embed font files.

### 3. Component Library, Not Page Designs
Build reusable pieces (buttons, cards, sections, navigation, footer, tables) that the site prototype will compose. Don't design specific pages here.

### 4. Mobile-Responsive by Default
Every component must work at 320px, 768px, and 1440px. Use CSS that adapts — no separate mobile stylesheets.

### 5. Accessibility Baseline
Minimum 4.5:1 contrast ratio for body text, 3:1 for large text (>18px or >14px bold). Semantic HTML in component examples. Keyboard-navigable interactive elements.

## Design System Outputs

All outputs go to `design-system/`:
- `identity.md` — Visual identity rationale
- `variables.css` — CSS custom properties
- `components.css` — Component styles
- `preview.html` — All components rendered for review
- `assets/` — Fonts, icons (self-contained)
