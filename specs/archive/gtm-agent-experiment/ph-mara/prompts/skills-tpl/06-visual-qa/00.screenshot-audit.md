# QA.00 Screenshot Audit

Prerequisite: Site (`site/`) and/or deck (`deck/`) exist. Preview server running.

## QA.00.1 Start preview server

- If not already running: `/preview-server-start --directory site`
- Capture the preview URL from the readiness output

## QA.00.2 Screenshot all pages

For each HTML page (index.html, subpages, deck/index.html):
1. Navigate to the page using playwright-cli
2. Wait `animationWait` milliseconds (from config, default 2000ms)
3. Take full-page screenshot at viewport width 375px → save as `screenshots/{page}-375px.png`
4. Take full-page screenshot at viewport width 768px → save as `screenshots/{page}-768px.png`
5. Take full-page screenshot at viewport width 1440px → save as `screenshots/{page}-1440px.png`

## QA.00.3 Review screenshots for issues

Check each screenshot for:
- **Layout**: Overlapping elements, horizontal overflow, broken grid alignment
- **Typography**: Orphans, widows, truncated text, unreadable font sizes
- **Contrast**: Text hard to read against background, low-contrast CTA buttons
- **Images/icons**: Missing, stretched, pixelated, wrong size
- **Navigation**: Menu overflow, broken links, CTA visibility
- **Spacing**: Inconsistent margins/padding, cramped or too-sparse sections
- **Responsive**: Elements that don't adapt properly between widths

## QA.00.4 Create issue log

Write `screenshots/issues.md`:
```
# Visual QA — Pass 1

## Issues Found

### Issue 1: {Description}
- Page: {filename}
- Width: {viewport}
- Screenshot: {filename.png}
- Severity: {critical/moderate/minor}
- Proposed fix: {CSS or HTML change}

## Summary
- Critical: {N}
- Moderate: {N}
- Minor: {N}
```

### Expected Scenario Outcome

`screenshots/` directory contains screenshots at all three widths for all pages. `screenshots/issues.md` lists all identified issues with severity and proposed fixes.
