# QA.01 Fix Issues

Prerequisite: Issue log from QA.00.

## QA.01.1 Prioritize

- Address critical issues first, then moderate, then minor
- Group related issues (e.g., all typography issues, all responsive issues)

## QA.01.2 Apply fixes

For each issue:
1. Identify the CSS or HTML change needed
2. Prefer CSS fixes over HTML restructuring
3. Prefer responsive CSS (`@media` queries) over hiding elements
4. Apply the fix
5. Consider: does this fix break other viewport widths?
6. Note the fix in `screenshots/issues.md`

## QA.01.3 Common fix patterns

- **Orphans**: `white-space: nowrap` on container, or adjust max-width
- **Overflow**: `overflow-x: hidden` on container, or `max-width: 100%` on child
- **Contrast**: Adjust color values in `variables.css`
- **Spacing**: Adjust spacing variables or add `@media` rules
- **Mobile nav**: Verify hamburger menu JS works, adjust breakpoint
- **Table collapse**: Verify card-stack responsive pattern at mobile width

### Expected Scenario Outcome

All identified issues have fixes applied. Each fix is documented in the issue log.
