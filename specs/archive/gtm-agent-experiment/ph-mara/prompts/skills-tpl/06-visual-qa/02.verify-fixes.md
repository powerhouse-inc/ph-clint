# QA.02 Verify Fixes

Prerequisite: Fixes applied in QA.01.

## QA.02.1 Re-screenshot

- Same process as QA.00: all pages at 375px, 768px, 1440px
- Save as `screenshots/{page}-{width}px-pass{N}.png` (incremented pass number)
- Wait for animations before each screenshot

## QA.02.2 Verify each fix

For each issue in the log:
- Is the issue resolved in the new screenshot?
- Update status in `screenshots/issues.md`: "Fixed" or "Partially fixed — {remaining issue}"

## QA.02.3 Check for regressions

- Compare new screenshots with old ones
- Did any fix introduce a NEW issue?
- If yes: add new issues to the log and repeat QA.01 → QA.02

## QA.02.4 Final sign-off

When all critical and moderate issues are resolved:
- Update `screenshots/issues.md` with final summary
- List any accepted minor issues with rationale
- Mark the QA cycle as complete

```
# Visual QA — Final

## Pass 1: {N} issues found
## Pass 2: {N} fixed, {N} new issues
## Pass 3: All critical/moderate resolved

## Remaining (accepted)
- {Minor issues with rationale for acceptance}

## Final Screenshots
- index-375px-pass3.png
- index-768px-pass3.png
- index-1440px-pass3.png
- ...
```

### Expected Scenario Outcome

All critical and moderate visual issues are resolved and verified. `screenshots/issues.md` has complete audit trail. HTML outputs are polished and responsive.
