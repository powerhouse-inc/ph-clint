# SP.00 Structure Pass

Prerequisite: Approved messaging and design system exist.

## SP.00.1 Read inputs

- Read `messaging/approved.md` — every section becomes an HTML section
- Read `design-system/` — use these component classes
- If either is missing, stop and tell the user which prerequisite skill to run

## SP.00.2 Create HTML skeleton

- Create `site/index.html` with:
  - `<!DOCTYPE html>`, proper `<head>` with meta tags and stylesheet links
  - Semantic elements: `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`
  - Section IDs matching messaging outline numbering
  - Component classes from design system
  - Placeholder text: `[Section N: Title]` for each section

## SP.00.3 Copy design system assets

- `site/css/variables.css` — from design system
- `site/css/components.css` — from design system
- `site/css/site.css` — new, page-level layout styles
- `site/assets/` — fonts, icons from design system

## SP.00.4 Create subpages (if needed)

- If messaging specifies multi-page structure, create subpage HTML files
- Each subpage gets the same `<head>`, nav, and footer
- Subpage content area has placeholder text

## SP.00.5 Verify skeleton

- `npx serve site/` should render the skeleton with styled components
- Every section from approved messaging has a corresponding HTML section
- Navigation links work (anchor links or subpage links)

### Expected Scenario Outcome

`site/` directory exists with HTML skeleton, CSS, and assets. All sections from approved messaging are represented. Serveable with `npx serve site/`.
