# Site Prototype - Skill Preamble

With this skill, you build the HTML website. The content is decided (`messaging/approved.md`). The visual system is decided (`design-system/`). Your job is to compose them into a serveable site.

## Interaction Mode

**expert mode**: Review each pass with the user before proceeding to the next. Present structure pass for approval, then copy pass, then trim pass.

**discovery mode**: Explain the three-pass process and why it separates content from design. Walk through decisions at each pass.

**one-shot mode**: Execute all three passes autonomously and present the finished site.

## Critical Guards

### 1. Three-Pass Process
This skill has three passes specifically to prevent mixing content and design decisions. Follow them in order:
1. **Structure pass** — HTML skeleton with semantic markup, section IDs, placeholder text
2. **Copy pass** — Fill in actual text from approved messaging
3. **Trim pass** — Cut excess, polish visual presentation, verify CTAs

### 2. No Content Changes During Production
The approved messaging is the spec. If you think something should be different, note it as a comment but implement what was approved. Content changes go back to skill 02.

### 3. Assets Within Serve Root
Copy design-system assets into `site/`. All paths must be relative. `npx serve site/` must work with no external dependencies.

### 4. Concise Copy in HTML
Some copy will need shortening for visual context (headlines, buttons, card descriptions). Shorten without changing meaning.

### 5. Funnel Links Throughout
Every 2-3 sections must have at least one CTA. Navigation includes a primary CTA button. Footer has conversion links.

### 6. Self-Contained Pages
Each HTML file includes all necessary CSS via shared stylesheet. No JS frameworks. Vanilla HTML/CSS with minimal vanilla JS for interactions (mobile menu, smooth scroll).

## Site Outputs

All outputs go to `site/`:
- `index.html` — Homepage
- `css/` — Stylesheets (variables, components, site layout)
- `assets/` — Fonts, icons
- `pages/` — Subpages (if multi-page structure)
