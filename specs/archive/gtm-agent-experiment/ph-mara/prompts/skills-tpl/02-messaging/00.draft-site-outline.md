# MS.00 Draft Site Outline

Prerequisite: Positioning brief in `research/` reviewed and approved.

## MS.00.1 Read the positioning brief

- Read all 5 files in `research/`
- Review `sources.json` for partner/ecosystem sites whose content should NOT be duplicated
- Internalize the emotional driver arc — this drives section order

## MS.00.2 Map drivers to sections

- Each emotional driver becomes a narrative beat in the site
- Define section hierarchy: hero → problem → solution → proof → details → conversion
- The hero addresses the most urgent driver
- Middle sections build trust through subsequent drivers
- Conversion sections appear after enough trust is established

## MS.00.3 Write section summaries

For each section, write:
- **Title**: The section heading (audience-facing)
- **Says**: 2-3 sentence summary of what this section communicates
- **Why here**: How this section serves the narrative arc
- **CTA**: What action the reader can take from this section (if applicable)

## MS.00.4 Define page structure

- Single-page with anchor links, or multi-page with subpages?
- If multi-page: what goes on the homepage vs. subpages?
- Rule: the homepage tells the complete story. Subpages provide depth.

## MS.00.5 Save as v1

- Save the outline as `messaging/v1.md`
- Include a changelog header: `## Changelog\n- v1: Initial draft based on positioning brief`

### Expected Scenario Outcome

`messaging/v1.md` exists with section-by-section outline, narrative justification, CTA placement, and page structure.
