# MS.01 Review & Iterate

This scenario repeats as many times as needed until the user approves the messaging.

## MS.01.1 Present for review

<!-- expert mode -->
Present the current version concisely. Highlight the key decisions and ask for specific feedback.

<!-- discovery mode -->
Walk through each section, explaining what it says and why. Ask if the audience identification, tone, and emphasis feel right.

<!-- one-shot mode -->
Self-review: does every section follow the guards? Does the narrative arc hold? Make improvements and present.

## MS.01.2 Process feedback

- Categorize the feedback: too technical? too long? wrong emphasis? missing angle?
- Map each piece of feedback to specific sections
- Identify if feedback requires changing the narrative arc (major) or just section content (minor)

## MS.01.3 Apply changes

- For each change: modify the section content
- Maintain narrative coherence — a change to one section may ripple to others
- Verify funnel distribution: are CTAs still well-placed after changes?
- Verify conciseness: is the outline getting shorter, not longer?

## MS.01.4 Log and save

- Add every change to the changelog with rationale
- Format: `- v{N}: {what changed} — {why, prompted by user feedback: "{feedback quote}"}`
- Save as `messaging/v{N}.md` (incremented version number)

## Convergence check

If changes are getting LARGER between versions instead of smaller, the positioning brief may need revision. Suggest going back to `/01-research` to revise the audience or driver analysis.

### Expected Scenario Outcome

New version `messaging/v{N}.md` saved with changelog documenting all changes and rationale.
