# MS.02 Finalize Messaging

Prerequisite: User has explicitly approved a version.

## MS.02.1 Confirm approval

- The user must explicitly approve a version ("v5 is approved", "this is good", "let's go with this")
- Do NOT finalize without explicit approval — even in one-shot mode, present for approval first

## MS.02.2 Create approved version

- Copy the approved version to `messaging/approved.md`
- Add an "Implementation Notes" section at the end with:
  - Any design/production considerations raised during iteration
  - Sections that need special visual treatment (forwardable, scannable tables, etc.)
  - CTA destinations that need to be confirmed

## MS.02.3 Summarize the iteration arc

- How many versions were produced?
- What were the major pivots? (e.g., "v1-v3: finding the right audience emphasis; v4-v6: trimming for conciseness")
- What's the final positioning in one sentence?

### Expected Scenario Outcome

`messaging/approved.md` exists as the final messaging blueprint. Ready for design system and site production.
