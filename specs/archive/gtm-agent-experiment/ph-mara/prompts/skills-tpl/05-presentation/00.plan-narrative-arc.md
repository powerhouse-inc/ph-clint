# PR.00 Plan Narrative Arc

Prerequisite: Research, approved messaging, and site prototype exist.

## PR.00.1 Read inputs

- Read `research/emotional-drivers.md` — the narrative arc drives slide order
- Read `research/audience.md` — tone and depth calibrated to audience
- Read `messaging/approved.md` — content drawn from approved messaging
- Review `site/` for screenshots to include

## PR.00.2 Map drivers to slides

For each emotional driver, plan 2-4 slides:
- **Driver slide**: State the concern in the audience's words
- **Response slide(s)**: Show how the product addresses it
- **Proof slide**: Evidence (screenshot, architecture diagram, concrete example)

## PR.00.3 Add framing slides

- **Opening** (1-2 slides): Who we are, what we do. Keep brief.
- **Bridge** (1 slide): "Here's what we're hearing from teams like yours" — transitions to driver arc
- **Closing** (1-2 slides): Next steps, CTA, contact information

## PR.00.4 Verify slide count

- Target: 15-25 slides for a 20-30 minute presentation
- If over 25: combine response slides or cut proof slides
- If under 15: are any drivers under-developed?

## PR.00.5 Save outline

Write `deck/outline.md` with slide title + 1-sentence purpose for each slide. Include which emotional driver each slide serves.

### Expected Scenario Outcome

`deck/outline.md` exists with slide-by-slide plan, narrative arc mapping, and target slide count.
