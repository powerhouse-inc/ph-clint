# PR.01 Produce Slides

Prerequisite: Slide outline (PR.00) complete.

## PR.01.1 Create HTML deck

- Create `deck/index.html` as a self-contained slide deck:
  - Each slide is a full-viewport `<section>` element
  - CSS `scroll-snap-type` for slide-like scrolling
  - Print styles with `page-break-after` for each section
  - Navigation: arrow keys or scroll to advance (minimal vanilla JS)
- Import design system variables for visual consistency

## PR.01.2 Write each slide

For each slide in the outline:
- **Headline**: 5-8 words, communicates the key point without reading further
- **Content**: Bullets, diagram, screenshot, or comparison — one type per slide
- **Speaker notes**: In `<aside class="notes">` — what the presenter would say (not visible in presentation mode)

## PR.01.3 Add visuals

- Take screenshots from `site/` using playwright-cli (or copy existing screenshots)
- Create simple SVG diagrams for architecture/flow concepts
- Embed all assets (inline SVG or relative paths within `deck/`)

## PR.01.4 Write deck CSS

- `deck/css/deck.css` imports design system variables
- Full-viewport sections with centered content
- Large readable type (minimum 24px for body, 36px for headlines)
- Slide number in corner (CSS counter)

## PR.01.5 Verify

- Open `deck/index.html` — navigate through all slides
- Does every slide make sense without a presenter?
- Are there any unsubstantiated metrics or claims?
- Do all screenshots render? Are SVG diagrams clear?
- Test at presentation resolution (1920x1080 or 1440x900)

### Expected Scenario Outcome

`deck/` directory contains a complete, navigable HTML slide deck following the emotional driver arc, with no fake metrics and all slides self-contained.
